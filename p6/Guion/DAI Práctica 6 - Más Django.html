Guión de Prácticas 6: Más Django

Introducción

En esta práctica pretendemos profundizar en un par de aspectos de Django: el uso de sus Modelos y el uso de plugins o complementos que ya existen para determinadas tareas.

Los modelos de Django

Lo modelos en Django son un tipo especial de objeto que se guarda en la BD. Los modelos definen cada tipo de dato que debe almacenarse en la BD y sus restricciones. En esta práctica vamos a familiarizarnos con la definición y uso de modelos. Siguiendo el tutorial de Django Girls crea en la aplicación restaurantes algún modelo. Por ejemplo, puedes crear un modelo que se llame Plato y que tenga algunos campos como por ejemplo nombre, tipoCocina ("Italiana", "Vegetariana", etc), alérgenos, precio... y asigna algunas restricciones (validadores) a cada campo que consideres oportuno (¿precios mayores que 0?).

Una ve readao el modelo, aplica los cambios en la base de datos de Django, importa el modelo para que pueda ser accesible desde el administrador (sigue el tutorial de Django girls) y prueba a añadir, borrar y editar objetos del nuevo tipo Plato. Intenta incumplir alguna restricción para que actue el mecanismo de validadores.

Formularios

Django incorpora un mecanismo para manejar los formularios de manera eficiente. En este apartado se pide crear algún formulario que le permita añadir editar o borrar Platos desde alguna de las páginas de su aplicación. Por supuesto, el formulario debe hacer todas las validaciones que sean pertinentes (evitar campos en blanco, valores incorrectos, etc).

Para nota: Usando modelos soportados por la base de datos de MongoDB (¡actualizado!)

Los modelos que se definen con Django usualmente se guardan en su propia base de datos. Sin embargo, en nuestra aplicación estamos usando información de una base de datos MongoDB. En este apartado de las prácticas (que es opcional) se pide investigar sobre las posibilidades de integrar los datos de la base de datos de MongoDB con un modelo similar a los que ofrece Django: Es decir, ¿podríamos acceder a los restaurantes de nuestra base de datos de igual manera a como accedemos a un objeto del modelo Plato que definimos anteriormente? Pista: MongoEngine.
